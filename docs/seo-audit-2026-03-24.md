## Critical

- **7 H1 tags on homepage — only 1 allowed** (Content)
    - Found: "Turn messy ideas into an organized plan", "Replace your scattered stack.", "From raw thinking to shipped work.", "Everything your project needs. One place.", "See it in action.", "You have three options.", "Start building context now."
    - Only the first should be H1. The rest should be H2.

- **No llms.txt file (404)** (AEO)
    - AI search engines (Perplexity, ChatGPT search, Claude) consume llms.txt for product context.
    - Create `/static/llms.txt` with product definition, features, pricing, target audience, and key page links.

- **No definitional "anchor sentence" in visible body text** (AEO)
    - When AI models are asked "What is BuildOS?", they need a clean extractable sentence in visible HTML.
    - Closest version is buried in JSON-LD and meta description, which crawlers weight lower than body text.
    - Add: _"BuildOS is an AI-powered productivity platform that transforms unstructured brain dumps into structured projects with tasks, documents, and a clear next step."_

- **Missing WebSite schema with SearchAction** (Structured Data)
    - Google uses this to power sitelinks search box in SERPs. Highest-value schema for a SaaS homepage.
    - Add as standalone JSON-LD block, ideally in `+layout.svelte` for site-wide coverage.

- **Missing standalone Organization schema on homepage** (Structured Data)
    - Organization exists only as nested "author" inside SoftwareApplication — minimal weight.
    - A full Organization block exists on `/about` but NOT on the homepage.
    - Use `@id: "https://build-os.com/#organization"` for entity linking across pages.

- **Logo image 250 KB PNG served at 40px — 60x overdelivery** (Performance)
    - `brain-bolt.png` is 512x446 intrinsic, displayed at 28-40px via Tailwind classes.
    - No width/height HTML attributes. Served as PNG, not WebP/AVIF.
    - A 40px WebP would be ~2-3 KB. Saves ~247 KB per page load.

- **7 render-blocking CSS files, 5 unused on landing page** (Performance)
    - `0.BKXseTkd.css` alone is 235 KB / 35 KB compressed (entire Tailwind + app CSS).
    - 5 component-specific files (Button, Modal, Textarea, PlanVisualization, AgentChatModal) are likely unused on the landing page.
    - All loaded as `<link rel="stylesheet">` with no media/async strategy.

## High Priority

- **No social proof / testimonials anywhere on landing page** (Content)
    - Zero user testimonials, case studies, review counts, user numbers, or logos.
    - Biggest E-E-A-T gap. Google quality rater guidelines look for "reputation of the website or content creator" via independent sources.

- **Heading hierarchy broken** (Content)
    - 7 H1s, ~15 H3 tags used as card titles under what should be H2 sections.
    - Proper structure: 1 H1 → multiple H2 sections → H3 subsections.

- **No visible author/founder attribution on landing page** (Content)
    - Meta author tag says "DJ Wayne" but zero visible author info, bio, or link to /about.
    - The /about page has strong E-E-A-T content (USMC Scout Sniper, 8 years software engineering, YC-backed) that is invisible to landing page visitors.

- **robots.txt lists "Claude-Web" — actual crawler is "ClaudeBot"** (AEO)
    - Add explicit entries for: `ClaudeBot`, `anthropic-ai`, `PerplexityBot`, `OAI-SearchBot`.

- **SoftwareApplication.offers price "0" — misleading** (Structured Data)
    - Google interprets price "0" as "free," but BuildOS is a paid product with a free trial.
    - Use `AggregateOffer` or multiple Offers showing actual paid price.

- **Missing aggregateRating** (Structured Data)
    - Primary trigger for SoftwareApplication rich results in Google. Without it, the schema provides minimal SERP enhancement.

- **3 duplicate viewport meta tags** (Technical)
    - Found in `app.html` and injected by Svelte components. The last one drops `viewport-fit=cover`.
    - Google flags duplicate viewports as an error. Consolidate to a single tag.

- **Sitemap stale — newest lastmod 45+ days old** (Technical)
    - Last updated 2026-02-07. New blog posts (e.g., `buildos-vs-chatgpt-context-that-compounds`) missing.
    - Should be dynamically generated or updated on each deploy.

- **400 KB compressed JS payload (1.28 MB raw) for unauthenticated visitors** (Performance)
    - Top chunks: d3/charting (144 KB compressed), markdown/text processing (101 KB compressed).
    - App-level dependencies loaded even for landing page visitors.

- **Static assets have no cache headers** (Performance)
    - `brain-bolt.png`, `s-brain-bolt.webp`, `favicon.ico` all return `cache-control: public, max-age=0, must-revalidate`.
    - Vercel header rules in `vercel.json` not being applied to `static/` folder assets.
    - Only `/_app/immutable/*` assets get proper immutable caching.

- **Pricing schema mismatch** (AEO)
    - JSON-LD on `/pricing` says $12.00 but the page displays $20/month. AI models will extract conflicting numbers.

## Medium Priority

- **Content-Security-Policy missing** (Security)
    - No protection against XSS injection or unauthorized resource loading.

- **X-Frame-Options missing** (Security)
    - Site can be embedded in iframes on any domain (clickjacking risk).

- **X-Content-Type-Options missing** (Security)
    - Browsers may MIME-sniff responses as scripts.

- **Referrer-Policy missing** (Security)
    - Full URLs (including query parameters) may leak to third parties.

- **Permissions-Policy missing** (Security)
    - Browser features (camera, mic, geolocation) not explicitly restricted.

- **HSTS missing includeSubDomains and preload directives** (Security)
    - Current: `max-age=63072000`. Recommended: `max-age=63072000; includeSubDomains; preload`.

- **Two User-agent: \* blocks in robots.txt** (Technical)
    - Per robots.txt spec (RFC 9309), directives from separate same-name groups are NOT merged.
    - The second block with `Crawl-delay: 1` may be ignored. Consolidate into one block.

- **Landing page paragraphs too short for AI citation** (AEO)
    - Average 15-30 words. Optimal for AI extraction: 80-200 words per paragraph.

- **Blog posts use narrative openings, not definitional leads** (AEO)
    - "I hated repeating myself..." vs. "BuildOS works by converting unstructured brain dumps into..."
    - First paragraph should directly answer the question implied by the title.

- **No datePublished or dateModified on landing page** (Content)
    - Only date signal is copyright "© 2026 BuildOS" in footer.
    - Add `dateModified` to JSON-LD for freshness signals.

- **Blog not surfaced on landing page** (Content)
    - 17 posts with topical authority across philosophy, productivity tips, getting-started guides.
    - Only 2 deep links to blog posts appear on the homepage.

- **sameAs in JSON-LD missing LinkedIn** (Structured Data)
    - Only Twitter listed. Footer links to LinkedIn (`linkedin.com/company/build-os-app`) — add to sameAs array.

- **Missing SoftwareApplication "image" property** (Structured Data)
    - "screenshot" is present but "image" is the preferred property for Google's processing.

- **CLS risk — logo images have no width/height attributes** (Performance)
    - Uses CSS-only sizing. Layout shifts when the 250 KB PNG loads before CSS parses.

- **Hero image preloaded but may not be the LCP element** (Performance)
    - `s-brain-bolt.webp` (38 KB) is preloaded but doesn't appear as a visible `<img>` on the landing page.
    - LCP candidate is likely the H1 text. Unnecessary preload wastes bandwidth priority.

- **CTA-to-content ratio high** (Content)
    - 6+ registration CTAs against relatively thin explanatory content.
    - Can signal a "thin affiliate" or "sales page" pattern to quality raters.
