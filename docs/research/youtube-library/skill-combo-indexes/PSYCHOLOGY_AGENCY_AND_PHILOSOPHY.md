**Canonical foundation (the doctrinal spine + the testable backbone + the counterweight)**

- [Ivan Illich — _Tools for Conviviality_ (analysis)](../analyses/2026-06-22_ivan-illich_tools-for-conviviality_analysis.md) — the convivial-tool test, the second watershed, "who commands whom." Doctrinal spine for "Designing tools that preserve agency." (Named directly by Ango in his interview.)
- [Deci & Ryan — Self-Determination Theory (analysis)](../analyses/2026-06-22_deci-ryan_self-determination-theory_analysis.md) — autonomy/competence/relatedness + crowding-out; turns "agency" from aesthetic to a pass/fail **Agency-Preserving Automation Test**.
- [Nicholas Carr — _The Glass Cage_ (analysis)](../analyses/2026-06-22_nicholas-carr_glass-cage_analysis.md) — automation paradox, deskilling, the generation effect; the **Deskilling-Risk Test**.

**Counter-school (cross-linked):** [Werner Jainek / Cultured Code — Things design philosophy](../analyses/2026-04-29_werner-jainek-cultured-code_things-design-philosophy_analysis.md) ([video](https://www.youtube.com/watch?v=XpI4sQybnm0)) — calm/opinionated software as the deliberate counter to malleability-maximalism.

## Internal BuildOS Use

Use this category to protect BuildOS from becoming a generic automation product. The sources are most valuable when they create guardrails around agency, taste, user control, and the difference between assistance and replacement. The three canonical tests now in the library — Illich's convivial-tool test, the SDT agency-preserving-automation test, and Carr's deskilling-risk test — can be run directly against any proposed BuildOS feature that automates part of the user's thinking.
