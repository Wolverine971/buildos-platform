## Why this draft

- **Aligned with `cold-outreach-rules.md`.** No mission-affirmation line, no "A bit about me:" transition, two explicit bridge sentences (one per thread).
- **Real connection leads: GovCon background + USMC.** Same opening structure as the GovEagle draft. The USMC line is intentionally understated — the fact of Marine service answers GovDash's "Care about America and her interests (non-negotiable)" requirement without being preachy. Don't tell them you care; let the credential carry it.
- **Second thread paraphrases their AI-product asks, then bridges.** Their listing wants "build interfaces that help users understand, guide, and trust AI-powered workflows … surface reasoning, highlight errors, emphasize citations, track progress." The BuildOS agentic-chat + SSE progress UI is the closest factual match. Dropped "citations" from the bridge because BuildOS doesn't do citation extraction — only claim what the credential supports.
- **Third paragraph is a stack credential, not a separate thread.** Curri shows up specifically for TypeScript / Node / React / Next.js at production scale + complex integrations (their nice-to-have). Doesn't need its own bridge — the closing line "is my daily driver at production scale" is the bridge.
- **No honest gap to disclose.** Their stack is DJ's daily driver. The rules say to name a gap if it exists; if it doesn't, skip the beat. Don't manufacture humility.
- **~230 words.** Same range as GovEagle v2 (210). Three threads each do real work.
