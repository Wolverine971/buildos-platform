# Worldbuilding Map — Reusable Template

**Purpose:** Run Caleb Ralston's "7 Elements of Worldbuilding" workflow against any project or brand (BuildOS, 9takes, Tacemus, DJ personal brand) and produce a worldbuilding map like [BUILDOS_WORLDBUILDING_MAP.md](BUILDOS_WORLDBUILDING_MAP.md).
**Source framework:** Ralston IG carousel [DaDw_8aGNhe](https://www.instagram.com/p/DaDw_8aGNhe/) — the 7 elements: Named Concepts, Re-defined Terms, Weaknesses, Characters & Settings, Interests & Values, Lore & Legends, Common Enemy.

## The workflow

1. **Harvest** — sweep the project's existing docs and extract raw material per element (coined phrases verbatim, enemy framings, lore beats, characters, settings, values). Don't invent yet; inventory first. Note which elements are oversupplied vs. empty.
2. **Draft the world in one paragraph** — the alignment statement everything hangs off: the place, the hero, the magic system, the villain, the founder's role. **Rule from the BuildOS run: hyper-specific founder lore is good, but the paragraph must hand the mirror to the reader** — the audience has to recognize their own version of the founder's story in it.
3. **Map each element** with these BuildOS-run conventions:
    - **Named Concepts:** tier them — _magic system / place / banner / ritual_ — and demote everything else to a bench. One concept per teaching. **Each core concept needs a word-hook AND an image-hook** (e.g. "context compounds" + the snowball). Oversupply with no hierarchy = nothing reaches "The Force" status.
    - **Re-defined Terms:** pick ≤4 redefinitions to repeat forever. Merge near-duplicates (BuildOS merged context+memory).
    - **Weaknesses:** flagship comeback pattern + a builder-confession lane (Seth Godin: publishing the flaw IS the work). Respect the Sensitive Index.
    - **Characters & Settings:** founder cast + **the audience archetype's own supporting cast** (a writer's editor/agent/publisher; a client's stakeholders). Settings should be renderable — feed them to `/moodboard`.
    - **Interests & Values:** values are usually documented; interests are usually locked away. Interests are cheap connection surface — green-light a metaphor bank.
    - **Lore & Legends:** origin legend, "before it was trendy" credibility beats, intellectual lineage (name the ancestors).
    - **Common Enemy:** expect villain oversupply. Collapse to **one apex villain** (two heads max); everything else becomes henchmen. Guardrails: punch systems not people; resolve into relief, not rage.
4. **Decision log** — split ratified ✅ vs. open 🔶; list the specific asks for DJ.
5. **Outputs** — the map itself, archetype maps for the project's 2–3 key personas, and a **content queue** (one file per post/blog idea generated along the way — see [content-queue/](content-queue/) for the format).
