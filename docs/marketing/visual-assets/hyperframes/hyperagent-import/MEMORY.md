# BuildOS Visual Director Memory

## Brand

- BuildOS is a thinking environment for people making complex things.
- Core promise: turn messy thinking into structured work.
- Differentiator: the project remembers.
- Moat: context compounds over time.
- Media doctrine: receipts over vibes, screens over scenes, work over spectacle.
- AI is the engine, not the marketing headline.
- BuildOS does not use generated product screens, generated founder footage, generated customers, or synthetic workspace scenes as proof.

## Visual language

- Inkprint feels like ink on paper, carved printmaking, field notes, and a printed dispatch from a workbench.
- Paper Studio colors: `#FBFAF7`, `#16161A`, `#66666E`, `#D6D0C6`, `#B85416`.
- Burnt orange is the only signal color and should be used once per frame.
- One dominant element per frame; calm hierarchy and negative space.
- Static texture represents overwhelm; Frame or Thread represents structure and memory.

## Production

- A production run begins with one real receipt and one claim.
- Default output is a 1080×1920 silent-readable Instagram Reel.
- The same composition may produce representative carousel frames and a cover candidate.
- Lint before render. Render one draft, stop for review, then one approved final.
- Add licensed music inside Instagram for organic publishing.
- Use only original or separately licensed royalty-free audio for reusable or boostable masters.

## Active campaign

- First campaign: Writer / Cold Manuscript.
- Public concepts: re-entry tax, Manuscript Memory, cold manuscript, keep the story warm.
- The draft remains in the writer's drafting tool. BuildOS holds the surrounding project state, open threads, decisions, and next moves.
- Do not claim that BuildOS replaces writing craft or drafting tools.
