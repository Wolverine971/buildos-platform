## Story decision

**Recognizable moment:** a project changes after the first brain dump and every collaborator needs the same updated state.
**One pain:** project context fragments across sessions and agents.
**One claim:** BuildOS can turn a natural-language capture into a shared project and build later updates on that project state.
**One CTA:** one shared project; keep moving forward.

## Receipt and boundary

- The Lantern House product screens are real seeded demo captures.
- The shown project, tasks, goal, plan, and document are demo data, not a customer result.
- The composition uses real product captures plus deterministic motion and explanatory graphics.
- It does not claim that BuildOS replaces the connected agents or makes every external agent integration automatic.
- No private workspace or customer data appears.
