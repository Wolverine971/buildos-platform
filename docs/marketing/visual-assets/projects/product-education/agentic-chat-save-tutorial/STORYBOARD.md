# BuildOS Brain Dump Tutorial — Storyboard

## Concept

**Message:** One natural brain dump becomes a durable shared project that can be updated tomorrow and read by connected agents.

**Format:** `1080×1920`, 36 seconds, portrait, 30 fps.

**Style:** BuildOS warm-paper editorial system from `DESIGN.md`; real `393×852` product captures inside uncropped iPhone frames.

**Rhythm:** `TYPE—ROUTE—ASSEMBLE / TYPE—UPDATE—CONFIRM / CONNECT—DRAW—HOLD`.

**Audio:** Silent first cut. `SCRIPT.md` contains narration for a voiced pass.

## Scene 1 — Brain dump → shared project (0:00–0:14)

- Headline: “Brain dump what’s going on with your project.”
- Left phone: real New Project agentic-chat modal. The podcast brain dump types directly inside the composer.
- Center: the animated Brain Bolt pulses with its dark field removed, then fades before a separate route arrow draws directly beneath it.
- Right phone: the real Lantern House project detail screen appears and locks beside the modal.
- Bottom: four creation receipts pop in—tasks, goal, plan, and document.
- Close: “Now you and the BuildOS agent are on the same page.”

## Scene 2 — Tomorrow’s update (0:13.4–0:25)

- Vertical push retains the same left-chat/right-project composition.
- Left phone: real Lantern House agentic-chat modal. “The guest list changed…” types in the same composer location.
- Center: the animated Brain Bolt pulses again, then fades before a separate route arrow draws directly beneath it.
- Right phone: Lantern House remains the visible shared project.
- The “Create launch checklist” task is outlined in Burnt Orange and labeled “Task updated.”
- Bottom: launch date, next tasks, and shared project state update receipts pop in.
- Close: “The new information builds on your previous work.”

## Scene 3 — Connected agents (0:24.4–0:36)

- Three official logo cards appear: OpenClaw, Claude, and ChatGPT.
- Burnt-orange SVG routes draw from each card into a central BuildOS Brain Bolt / Lantern House project hub.
- The hub identifies Tasks, Goal, Plan, and Document as shared project context.
- One final route draws downward to: “One shared project. Keep moving forward.”
- Hold the final composition long enough to read without narration.

## Truth and privacy boundaries

- All product screenshots are real seeded demo captures.
- Screens remain at native mobile aspect ratio and are never cropped horizontally.
- Receipt labels describe the visible seeded Lantern House entities.
- No unrelated workspace data or private account information appears.
