# Cold Manuscript — HyperFrames pilot storyboard

**Format:** 9:16 Reel · 1080×1920 · 30 fps · 16.5 seconds
**Lane:** real-media only · no generated imagery · silent-readable
**Purpose:** prove that one BuildOS story system can produce both a static carousel and a motion derivative.

| Time       | Story job                  | Visual                                                                           |
| ---------- | -------------------------- | -------------------------------------------------------------------------------- |
| 0.0–3.4s   | Name the world             | Paper Studio hook: “You don't reopen a blank page.”                              |
| 3.2–6.4s   | Make the pain felt         | Ink Room interruption: 90 minutes available, 40 lost to reconstruction           |
| 6.2–10.3s  | Show the invisible burden  | Existing Inkprint “muddy middle” card                                            |
| 10.1–13.9s | Provide the receipt        | Existing real BuildOS before/after asset                                         |
| 13.7–16.5s | State the product boundary | The draft stays in the writing tool; BuildOS holds the project context around it |

## Carousel derivative

Use the five scenes as slides, with the receipt scene split into two slides if a six-card carousel performs better:

1. You don't reopen a blank page. You reopen a cold manuscript.
2. You finally get 90 minutes to write. Forty disappear remembering the book.
3. “Your manuscript is not the only thing you're holding.” (`asset2-painlist-light.png`)
4. Name it: the re-entry tax.
5. Real brain dump → real BuildOS project (`asset1-beforeafter-light.png`).
6. The draft lives in the writing tool. BuildOS holds the project context around it. Ask: “Where does your book's context live?”

## Quality rules

- The real BuildOS receipt is the proof and must remain legible enough to register as a real interface.
- Burnt orange is the only signal color.
- Keep the piece silent-readable. Add real founder voiceover later only if it improves the story.
- Do not add generated desk scenes, fake people, AI UI, stock “author typing” footage, or generic productivity b-roll.
- Do not claim automatic story-bible generation, “Memory Mode,” or a five-minute resume time.
