## File Types

### Campaign brief

Format: `YYYY-MM-DD_<campaign-slug>_campaign.md`

Defines: segment choice, pain thesis, proof asset, CTA, channel plan, scorecard, next 7 actions.

### Execution checklist

Format: `YYYY-MM-DD_<campaign-slug>_checklist.md`

Day-by-day task list so the campaign can't silently drift.

### Target list research handoff

Format: `YYYY-MM-DD_<campaign-slug>_target-list_handoff.md`

The brief handed to a research agent (or the founder) to go find the named targets. Defines buckets, criteria, and search strategy.

### Target list

Format: `YYYY-MM-DD_<campaign-slug>_target-list.md`

The actual 20 people with profile URLs, pain signals, and bucket classification. Deliverable of the handoff.

### Post / outreach drafts

Format: `YYYY-MM-DD_<campaign-slug>_<channel>-post-drafts.md` or `_outreach-drafts.md`

Pre-written hooks, post drafts, DM templates, and comment reply guides for the channels the campaign uses.

### Campaign check-in

Format: `YYYY-MM-DD_<campaign-slug>_checkin.md`

Weekly progress, metrics, learnings, verdict, next actions.

---

## Rules

- Prefer one active campaign at a time.
- Two active campaigns is the hard maximum.
- Every campaign must have one primary segment and one primary CTA.
- Every campaign must end with a verdict:
    - continue
    - narrow
    - pivot
    - kill
- Every campaign folder must cross-link its files so research does not get lost between the brief, the handoff, the target list, and the check-in.
