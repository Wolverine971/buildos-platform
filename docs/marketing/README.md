## Content creation flow

```text
campaign decision
  -> real receipt / source material
  -> asset plan or storyboard
  -> visual-assets/projects/<category>/<project>/
  -> delivery/ master + source package
  -> publish
  -> campaign scorecard + relationship follow-up
```

The reusable HyperFrames workflow, templates, and setup live in [`visual-assets/hyperframes/`](visual-assets/hyperframes/README.md). Actual compositions and their masters live in [`visual-assets/projects/`](visual-assets/projects/README.md).

## Source-of-truth rules

- Use this page as the folder map; do not create another marketing index.
- Use `START_HERE.md` for current orientation and `INDEX.md` for the broad archive catalog.
- Keep campaign strategy with its campaign. Keep platform logs and person memory in `social-media/`.
- Keep taskers as execution history and handoffs. Link them to the canonical marketing document instead of duplicating the plan.
- Commit authored documents, project manifests, composition source, curated final masters, and compact source packages.
- Keep raw captures, copied media caches, thumbnails, snapshots, QA frames, and draft/timestamped renders local and ignored.
