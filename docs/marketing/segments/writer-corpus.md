# Writer Corpus — Hub

Everything BuildOS knows about **writers/authors** lives across four folders (`brand/worldbuilding/`, `segments/`, `visual-assets/`, `growth/`). This is the one door in. Each doc has **one job**; use the canonical owner for that job and don't re-derive it elsewhere.

> **The finding that shaped this hub (2026-07-06):** the writer docs aren't redundant — they're deep on **opposite axes**. The May segment docs carry the _evidence_ (tools, prices, ops pipeline, sourced quotes, outreach targets); the July worldbuilding docs carry the _voice_ (re-entry ritual, drawer novel, product surfaces, AI-trust boundary). The roll-up layers voice **on top of** evidence and reconciles one positioning conflict — it doesn't delete anything.

## Positioning ruling — memory hero, integration demoted (2026-07-06)

The single strategic decision the whole corpus now obeys:

- **Lead with manuscript memory and relief.** _"BuildOS never writes your book. It remembers it."_ The words stay the writer's; BuildOS never generates prose. This disarms the segment's (strong, well-documented) anti-generative-AI reflex before selling anything.
- **Demote the integration story.** "Your other tools can read from BuildOS" survives only as a quiet **Author-Operator** power-user capability — the writer's choice, on their terms. It is never the hero pitch, and never framed as "make your AI co-writer better."
- Superseded framing (do not reuse): _"so ChatGPT, Claude, and Sudowrite all read from the same brain"_ as a headline.

## Route by what you're doing

- **Writing a writer blog post / social / headline** → start Tier 1 (archetype voice) → pull evidence from Tier 3. First series to draft: the **"Cold Manuscript" series** scoped in `CORE_SYNTHESIS.md`.
- **Writing landing/product copy** → Tier 1 for the voice + the memory-hero ruling above → Tier 2 TL;DR/hero line for the exact tested language.
- **Cold outreach to an author** → Tier 2 outreach shortlist (ranked first-wave picks) + Tier 3 §6 candidates → verify send-status against the current skip list first.
- **Naming a product surface for writers** → Tier 1 "How BuildOS shows up" (Manuscript Home, Re-entry Brief, Scene Queue, Memory Mode) + `07-buildos-translation-opportunities.md` for the full surface list.
- **Building a visual asset** → Tier 4 teardown asset plan (re-point to memory-hero first — see gaps).
- **Quoting a stat, price, or community size** → Tier 3 May research **only after** a freshness check (its numbers are point-in-time May 2026).
