# The Writer's AI-Age Flow

_A one-page method for staying prolific. Print it. Keep it by your desk._

> **The prolific writers aren't generating more. They're losing less.**
> You don't have a capture problem. You have a retrieval problem.

---

## The 6-step flow

**1. Capture frictionlessly — into ONE place.**
The cost of capturing has to be lower than the thought is fleeting. One inbox, not five apps. Don't decide where it goes yet. Just get it out of your head.

**2. Sort once, deliberately — and later.**
Separate sorting from capturing. Let notes sit. Process only the survivors — the ideas that still pull after a few days. Organize by **theme**, not by source. Pay the sorting cost up front.

**3. Keep one project brain.**
Research, decisions, character/continuity details, and open threads live in one connected place that _remembers_ — not in your head, not scattered across tools.

**4. Route across AI surfaces without re-pasting.**
Use the best model for each job — research, drafting, editing — but let them all read from the same project brain. Context lives **outside** any one chat, so you never rebuild it. (Remember: curated context beats a giant paste. More tokens can make the model _worse_.)

**5. Draft fast from pre-sorted material.**
When the sorting's done, drafting is assembly. Write with the door closed. Type **"TK"** where a fact is missing and keep moving — never research mid-draft.

**6. Revise with distance.**
Put the draft down. Come back a near-stranger. _Now_ cut your darlings.

---

## The counterintuitive rules (steal these)

- **Stop when it's going well**, not when you're stuck — leave momentum loaded for tomorrow (Hemingway, Doctorow).
- **Aim absurdly low.** A page a day beats a heroic burst. Small quotas survive bad days (Doctorow, Trollope). Don't even _beat_ your quota — it poisons the next day (King).
- **Wait before processing notes.** Only what still matters in a week earns a permanent place (Holiday).
- **Don't research while drafting.** Phase it. "TK" and move on (McPhee, Doctorow).
- **Don't start top-down.** Let the draft assemble from notes you already wrote (Zettelkasten).
- **Write fast, edit slow.** Speed keeps the world alive; distance makes it good (King).
- **The method beats the app.** McPhee uses a 1980s editor; Sanderson uses an open-source wiki. Start today with whatever you have.

---

## The one line to remember

**The AI isn't the bottleneck. The workflow is. Be prolific by losing less.**
