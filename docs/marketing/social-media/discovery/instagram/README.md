# Instagram Discovery

Instagram discovery is Stage 0 of the BuildOS relationship pipeline.

It exists to keep `/instagram-warmup` from relying only on the same known accounts. It should continually surface new creator-founders, PKM operators, writers, course creators, freelancers, and high-quality watering-hole commenters.

## Files

- `search-terms.md` - active, testing, retired, and skipped search terms
- `candidates.md` - current candidate queue and handoff state
- `YYYY-MM-DD_discovery.md` - optional dated discovery notes for substantial scans

## State Flow

`new -> profiled -> queued_for_warmup -> engaged`

Side exits:

- `monitor` - worth watching, no action now
- `skip` - disqualified or low-quality source

## Rules

- Discovery never comments, likes, follows, or DMs.
- Every queued candidate needs a specific post, source, or comment thread to inspect.
- Prefer peer-tier accounts before mega accounts.
- Use mega accounts as watering holes, not direct targets.
- Capture skip reasons so the next run does less repeated work.
- Keep ADHD as a supporting lane unless there is a direct relationship signal.
