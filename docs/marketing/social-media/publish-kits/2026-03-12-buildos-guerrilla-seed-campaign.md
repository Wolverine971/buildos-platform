# BuildOS 6-Week Guerrilla Seed Campaign

## Purpose

This is the current 4-6 week campaign plan for BuildOS.

It is built for a one-person team.

The goal is not to create perfect finished copy in advance.
The goal is to create strong seeds that can be turned into posts without starting from zero.

---

## Campaign Goal

Over 6 weeks, install one clear market association:

**BuildOS is a thinking environment that turns messy thinking into structured work.**

The campaign should also make two things feel true:

- this can replace scattered tools
- the project remembers instead of forcing me to remember everything

---

## Operating Rules

### Rule 1

Each week gets:

- one thesis
- one audience
- one hero proof asset

### Rule 2

Use the same thesis across LinkedIn, X, and Instagram.
Do not invent a new angle for each platform.

### Rule 3

Seed first, polish second.

For every asset, generate:

- 3-5 hook options
- 2-3 demo options
- 2 LinkedIn angles
- 2-3 X angles
- 2 Instagram angles
- 2 CTA options

### Rule 4

If time is tight, ship in this order:

1. hero proof asset
2. X post
3. LinkedIn post
4. Instagram carousel or reel

---

## Weekly Rhythm

### Monday

Capture or produce the hero proof asset.

### Tuesday

Publish the clearest X post and/or short demo.

### Wednesday

Publish the LinkedIn angle.

### Thursday

Publish the Instagram carousel or reel adaptation.

### Friday

Publish founder proof, a follow-up contrast post, or a second X variation.

This is the minimum viable rhythm, not a prison.

---

## Weeks 1-4: Core Campaign

These four weeks are the core.
If energy is low, complete these first.

## Week 1

### Thesis

Creators do not need another tool.
They need a thinking environment.

### Audience

Broad creator-builder audience

### Hero Proof Asset

A fragmented workflow turning into one connected BuildOS project.

### Hook Seeds

- most creator workflows are five disconnected tools pretending to be a system
- the problem is not lack of ideas. it is fragmented context
- a blank doc is not a thinking environment
- notes + docs + chat + tasks everywhere is not a workflow
- BuildOS is where the project starts holding together

### Demo Options

1. notes app + Google Doc + ChatGPT + checklist -> one BuildOS project
2. raw brain dump -> structured project + next move
3. screenshot of connected assets inside one creator project

### LinkedIn Angles

1. Why "productivity tool" is the wrong category for creators with long workflows
2. The real problem is fragmented context, not lack of discipline

### X Angles

1. short anti-tool-sprawl post
2. before/after workflow reality check
3. "that is not a system" contrast post

### Instagram Angles

1. carousel: `your project is living in 5 places`
2. reel: messy inputs becoming one project with a visible next step

### CTA Seeds

- if your work is spread across too many places, that is the problem
- if you are building something complex and the context keeps slipping, this is for you

## Week 4

### Thesis

Replacement beats addition.
Stop adding tabs. Replace the stack.

### Audience

Creators and founders already tired of tool sprawl

### Hero Proof Asset

A scattered stack comparison card or short demo showing replacement.

### Hook Seeds

- most people do not need another AI tool. they need fewer places to think
- waiting for the perfect AI tool is still a workflow decision
- if you need four apps and three chats to move one project forward, your stack is the problem
- one working system beats five half-systems
- replacement is the pitch, not addition

### Demo Options

1. Notes + Trello + ChatGPT + Google Docs -> BuildOS
2. voice notes + random docs + task list -> BuildOS
3. return-to-project demo showing restart friction disappearing

### LinkedIn Angles

1. The market is saturated with AI promises because almost nobody is selling relief
2. Why replacement framing is stronger than "add BuildOS to your workflow"

### X Angles

1. anti-AI-fatigue post
2. "keep juggling tools / wait for the perfect app / start building context now"
3. stack comparison graphic

### Instagram Angles

1. carousel: `your stack is the problem`
2. reel: what it feels like when the work finally lives in one place

### CTA Seeds

- if you are allergic to "one more app," good. that is exactly the right instinct
- if your current workflow only works when you remember everything, replace that first

## Reusable Seed Bank

### Hook Starters

- the project should not disappear the second you close the tab
- a blank doc is not a workflow
- your brain is not broken. your stack is fragmented
- one place to think beats five places to remember
- the right tool makes restart easier

### Contrast Seeds

- notes app vs thinking environment
- blank document vs living project
- stateless chat vs project memory
- productivity theater vs actual forward motion
- more apps vs one system

### CTA Seeds

- if this looks like your workflow, you are the audience
- if your work is complex and the context keeps scattering, follow along
- if you are tired of rebuilding the same project from scratch, that is the pain BuildOS is built for

## What Success Looks Like

By the end of this campaign, the desired response is not just:

- cool tool

It is:

- this is exactly my workflow
- this feels different from the usual AI stuff
- can this replace what I am currently using?
- I finally understand what BuildOS is

That is the standard.
