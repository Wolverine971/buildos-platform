# Tom — Wolves of Glendale (comedy band) + Iron Songwriter

## Profile

- **Personal short-form:** Comedy songs (Cream Cheese, Hot Dogs — viral 45-60s formats).
- **Band:** **Wolves of Glendale** — comedy music band. Plays live shows (one C&S-noted: he screened the pilot for the band's live audience and got "huge laughs").
- **Format being launched:** **Iron Songwriter** — time-constrained songwriting challenge. Pilot uses Shrek dialogue lyrics, must upload to Spotify when done.
- **Cadence target:** Long-form show.
- **Distribution:** Instagram-native shareables transitioning to YouTube long-form.

## What he's building

Iron Songwriter as a long-form YouTube show. Pilot episode is done (Shrek-lyric song). Needs help with packaging — title, thumbnail, hook. C&S did a deep workup: added genre wheel, mystery instrument box, FaceTime guests, progress checklist, real crowd footage to add stakes.

## The on-air problem (verbatim)

> "We're now transitioning to long form with the band. I'm producing this show. I've cut the first episode of the show. The problem is how to package it correctly. Title thumbnailing — very new territory to me."

> "I think the cut is really fun. We even screened the cut at a Wolves show live and it was getting huge laughs in the room which was so exciting."

## Colin & Samir's diagnosis

1. Confirm-click in 7s, make-personal at 11s, new open loop by 30s.
2. "Lines" not "dialogue" in the title — clunky word choice.
3. Add adversarial variables — genre wheel, mystery instrument box, time pressure they don't control.
4. Show real crowd footage early — "real band, real stakes."
5. Title: "we made a song using only" framework.
6. Thumbnail: one face not three (until they're famous enough for the band image).
7. Build the engine: every episode follows the same checklist + reveal structure.

## Tool / workflow signal

- **Band = multi-person stakeholder.** Show is co-produced. Multiple people consume the same scripts / cuts.
- **Comedy + music = unique production stack.** Music recording, Spotify upload, video editing all in one episode.
- **Live show calendar** — bookable real-world events plus YouTube + IG content. Three pipelines.
- **Engine format being designed in real-time on C&S's whiteboard** — checklist, reveals, mystery box, FaceTime guest.

## BuildOS angle

Iron Songwriter is a multi-pipeline project (long-form YT show + band + live shows + Spotify uploads of each song + short-form clips per episode). The format itself is a recipe with reveals + adversarial variables + a checklist + a FaceTime guest. That recipe is the engine and it needs to live somewhere. Each episode has a guest, a genre, a movie / theme, a song delivered to Spotify, a video on YouTube, a short on IG. **Each episode = one project with 5 surfaces.**

**Pitch shape:** "Iron Songwriter is a five-surface show — long-form YouTube, IG shorts, a Spotify upload, a band-show callback, and the next episode's references. The recipe (checklist + mystery box + FaceTime guest + adversarial variables) is your engine. BuildOS holds one project per episode so the next episode reads from what landed in the last one. The band's calendar lives next to the show's pipeline."

## Risk / fit notes

- Multi-person team (band). Buyer is Tom but the system needs to serve the others.
- "Iron Songwriter" already has a Subway-Takes-style structure (host + guest + constraint). Strong format = high BuildOS leverage.
- Wolves of Glendale Spotify presence — the music distribution side is unique vs other guests. Don't oversell, but signal that BuildOS holds the Spotify + YouTube + show calendar as one thing.
