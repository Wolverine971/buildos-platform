# Marketing Site Design Review

Use this child when the surface is selling, positioning, explaining, or converting rather than helping an existing user complete an app task.

## When to Use

- Reviewing a landing page, homepage, product page, launch page, portfolio, case-study page, comparison page, or pricing page.
- The user asks whether a page communicates value, looks credible, converts, or has a strong hero and CTA path.
- A general UI review would miss offer clarity, proof, trust, and section-level conversion issues.

## Workflow

1. Identify the page goal, audience, promise, primary CTA, and proof standard.
2. Review the first viewport for literal offer clarity, product/category signal, credibility, CTA priority, and next-section affordance.
3. Review section sequence: problem, outcome, mechanism, product evidence, proof, objections, pricing, FAQ, and final CTA.
4. Apply visual fundamentals: hierarchy, spacing, type roles, color discipline, media quality, section rhythm, and mobile fit.
5. Return a section-by-section scorecard with highest-impact copy, layout, proof, and visual changes.

## Guardrails

- Do not judge a marketing page only as an app UI. Selling pages need value clarity and proof.
- Do not recommend vague hero copy. The visitor should understand the category and concrete offer quickly.
- Keep visual polish tied to conversion trust, not decoration.
