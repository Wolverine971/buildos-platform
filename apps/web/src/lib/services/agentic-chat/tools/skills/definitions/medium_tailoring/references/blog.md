# Blog Tailoring

Reshape an approved draft into a blog post / article. A blog carries long text + media and is discovered by search and by links, so structure, scannability, and an honest on-page SEO surface matter. (Strategy — whether a blog is even the right format for the intent — belongs to `content_strategy_beyond_blogging`; this reference assumes the blog is chosen.)

## Format constraints

- Renders full markdown: headers, lists, images, code, blockquotes — use them for scannability.
- No length cap, but earn every paragraph. Typical strong post: **800–2,000 words** depending on depth.
- Above-the-fold (title + first paragraph) is the attention slot for both readers and search snippets.

## Structure template

```
Title (H1)        outcome- or curiosity-led, ≤ ~60 chars for clean SERP display
Intro (1–2 ¶)     answer the reader's immediate question fast; no long wind-up
Body              H2/H3 sections, one idea per section, short paragraphs
  - bullets/numbered lists for steps
  - one image/diagram per major claim (carry sensory_double_tap cues here)
  - blockquote the single most quotable line
Takeaway          a clear reframe or summary
CTA               one next step
```

## Scannability

- Section headers a skimmer can read alone and still get the argument.
- Paragraphs **2–4 sentences**; break walls.
- Front-load the payoff (goal completion beats dwell time) — do not save the point for the end.

## On-page SEO surface (honest, not stuffed)

- Put the **topic noun in the first 5–7 words** of the title and in the intro.
- Use descriptive H2s that match how people phrase the question.
- One clear meta description (~150 chars) that states the payoff.
- **Internal links**: link to ≥2 related pieces; this is the most reliable durable-distribution lever.
- Do not keyword-stuff or fabricate; write for the reader first.

## Thresholds

- Title ≤ ~60 chars, topic noun early.
- Paragraphs 2–4 sentences.
- ≥2 internal links; one primary CTA.

## Amplification levers

- Internal links to build a hub/cluster (durable distribution that does not depend on a feed).
- One quotable blockquote that travels when shared.
- A diagram or worked example at the key claim (dwell + comprehension).

## Common mistakes

- A long personal wind-up before answering the question.
- Wall-of-text with no headers or lists.
- Title that buries the topic noun or exceeds the SERP cutoff.
- No internal links (orphan post).
- Keyword stuffing that reads as written-for-robots.
