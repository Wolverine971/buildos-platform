# Daily Briefs Feature

Ontology daily briefs are generated by the worker, stored in `ontology_daily_briefs` and `ontology_project_briefs`, and delivered through the notification pipeline. The legacy `daily_briefs` / `project_daily_briefs` flow is not the active worker path.

## Active Flow

1. `apps/worker/src/scheduler.ts` runs hourly and reads active `user_brief_preferences`.
2. The scheduler uses the user timezone from `users.timezone`, calculates the next notification time, and queues generation with a 2-minute pre-generation buffer.
3. When `ENGAGEMENT_BACKOFF_ENABLED=true`, `apps/worker/src/lib/briefBackoffCalculator.ts` decides whether inactive users should receive a standard, re-engagement, dormant, or skipped brief.
4. The Supabase queue stores a `generate_daily_brief` job in `queue_jobs`.
5. `apps/worker/src/worker.ts` routes `generate_daily_brief` jobs to `apps/worker/src/workers/brief/briefWorker.ts`.
6. `briefWorker.ts` validates timezone/date data, skips stale scheduled jobs, and calls `generateOntologyDailyBrief(...)`.
7. `apps/worker/src/workers/brief/ontologyBriefGenerator.ts` loads ontology data, generates project briefs, renders the main brief, and writes `ontology_daily_briefs` plus `ontology_project_briefs`.
8. `briefWorker.ts` emits a `brief.completed` notification event; email/SMS delivery is handled by the notification adapters. If audio is enabled, it also enqueues `generate_brief_audio`.

## Current Behavior Notes

- Canonical storage is `ontology_daily_briefs` and `ontology_project_briefs`; do not document new worker behavior against `daily_briefs` / `project_daily_briefs`.
- Calendar data is currently loaded from BuildOS-known internal rows, not a live Google Calendar read at brief generation time.
- Calendar prompts receive compact counts and at most a few commitments, not raw calendar dumps.
- `ENGAGEMENT_BACKOFF_ENABLED` defaults off unless the worker environment sets it to `true`.
- Brief notifications are sent by the notification system after generation, not by the generation worker directly.
