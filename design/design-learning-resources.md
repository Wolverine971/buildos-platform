## Core Concepts to Study

### From Ryo Lu's Philosophy

| Concept                     | What It Means                     | How to Apply                    |
| --------------------------- | --------------------------------- | ------------------------------- |
| **Systems over Features**   | Unify, don't fragment             | Look for patterns that combine  |
| **Simplicity ≠ Minimalism** | Simple cores, emergent complexity | Build flexible primitives       |
| **Designing Containers**    | Design systems, not screens       | Think about configuration space |
| **Slop → Soul**             | AI output is the beginning        | Iterate and inject craft        |
| **Feel It**                 | Prototype in code, not mocks      | Build to understand             |

### Design Systems Thinking

| Concept                     | Source         | Study This                    |
| --------------------------- | -------------- | ----------------------------- |
| **Atomic Design**           | Brad Frost     | Atoms → Molecules → Organisms |
| **Design Tokens**           | Salesforce     | Abstracting design decisions  |
| **Compound Components**     | React patterns | Flexible, composable UI       |
| **Constraint-based Design** | Various        | Using limitations creatively  |

### AI Interface Patterns

| Concept                     | What It Means                 | Examples                      |
| --------------------------- | ----------------------------- | ----------------------------- |
| **Progressive Disclosure**  | Reveal complexity gradually   | Cursor's agent modes          |
| **Graceful Degradation**    | Work at all capability levels | Manual ↔ Automated spectrum  |
| **Instrumental vs Engaged** | Task completion vs mastery    | GPS vs Maps (Linus Lee)       |
| **Human-in-the-Loop**       | Strategic human placement     | Review points in AI workflows |

## Learning Path Suggestions

### If You Want to Think Like Ryo Lu

1. **Start:** Watch all Ryo Lu interviews (you have transcripts)
2. **Study:** Notion's design philosophy (block-based thinking)
3. **Practice:** Build something with Cursor, pure vibes
4. **Read:** Linus Lee on tools for thought
5. **Apply:** Audit your own products with systems lens

### If You Want to Build AI Interfaces

1. **Start:** Use Cursor daily, notice what works
2. **Study:** Latent Space podcast for technical depth
3. **Practice:** Build a chat interface, iterate on UX
4. **Read:** Anthropic's research on AI UX
5. **Apply:** Focus on "feel" - prototype in code

### If You Want Design Systems Mastery

1. **Start:** Study Linear's design (simple but powerful)
2. **Study:** Atomic Design, Design Tokens
3. **Practice:** Build a small design system
4. **Read:** Brad Frost, Nathan Curtis
5. **Apply:** Refactor existing UI into system
