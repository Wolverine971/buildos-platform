## Receipt line format

One line per receipt. No multi-paragraph entries. If you need more than 200 characters to describe it, paste the source link and summarize in one sentence.

```
- {YYYY-MM-DD} · {one-line claim or quote in quotation marks} · source: {name/outlet}, {url} · supports: {term-to-own, topic cluster, or T##}
```

Example:

```
- 2026-04-16 · One streamer generated 2.2B views via 69,000 clips in 30 days · source: Devin Nash, https://youtube.com/watch?v=-5lqJF5BTvo · supports: "interest media" / T34 / T40
```

## Rules

1. **Always timestamp.** Receipts age fast; a 2-month-old campaign reads differently than a fresh one.
2. **Name the source.** "Studies show" is not a receipt — it's the absence of one.
3. **Link the source.** URL required unless it's offline (book, paper). If offline, name the page / chapter.
4. **Tag what it supports.** A term-to-own, a topic-map cluster, or a specific T##. If unsure, tag the term only.
5. **Don't dedupe.** If something is a receipt for both the paid-clipping section and the creator-data section, add it to both. Redundancy helps retrieval.

## When to NOT capture

- Generic "the algorithm is bad" takes with no specific mechanism or number
- Opinion posts that don't cite primary evidence
- Your own past tweets (use the cluster posts themselves, not your tweets)

## Hand-off

After adding receipts, don't update any dashboards. T45 is deliberately lightweight — the value is in fast capture, not in status tracking.
