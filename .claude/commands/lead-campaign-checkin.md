# Lead Campaign Check-In

You are running a **weekly campaign review** for an active BuildOS lead-generation campaign.

This command is for the **progress + learning + next-steps** pass.

Do not rewrite the entire strategy.
Do not hide behind soft language.
Call the result clearly: working, mixed, stuck, or failing.

---

## Output

Create a dated check-in doc at:

`docs/marketing/growth/lead-campaigns/YYYY-MM-DD_<campaign-slug>_checkin.md`

Read the active campaign brief first.
If prior check-ins exist for the same campaign, read the most recent one before writing the new one.

---

## Read These First

- `/docs/marketing/growth/lead-gen-operating-system-2026-04-10.md`
- `/docs/marketing/growth/lead-campaigns/README.md`
- the active campaign brief for the selected campaign
- the latest prior check-in for that campaign, if any
- relevant warmup docs during the check-in window
- relevant reply or outreach docs during the check-in window
- any linked performance docs or notes created during the campaign

Use the `legion-lead-strategist` mindset.

---

## What To Review

Review the campaign across five layers:

1. activity
2. attention
3. response
4. qualified traffic
5. activation or activation-proxy behavior

If a layer has no measurable data, say that explicitly and note how to capture it next time.

---

## Required Sections

1. `Status`
   - `working`
   - `mixed`
   - `stuck`
   - `failing`

2. `What Shipped`
3. `What Happened`
4. `Metrics`
5. `What We Learned`
6. `What We Should Stop`
7. `What We Should Double Down On`
8. `Next 7 Actions`
9. `Campaign Verdict`
   - continue
   - narrow
   - pivot
   - kill

---

## Review Rules

- Separate effort from outcome.
- Separate raw traffic from qualified traffic.
- Separate signups from activation.
- If the campaign created meaningful conversations or strong proof even with low traffic, record that honestly.
- If the campaign produced activity but no movement, call that out.
