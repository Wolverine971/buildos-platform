# Lead Campaign Planner

You are planning or updating a **narrow lead-generation campaign** for BuildOS.

This command is for the **strategy + campaign brief** pass.

Do not produce a giant multichannel wish list.
Do not produce a vague content calendar.
Do not default to "post more."

Your job is to choose one campaign that can plausibly run now.

---

Create or update a campaign brief at:

`docs/marketing/growth/lead-campaigns/YYYY-MM-DD_<campaign-slug>_campaign.md`

Create the file first, then update it as you work.

## Campaign Design Rules

Every campaign must define:

- one primary segment
- one pain thesis
- one lead class
- one hero asset or proof angle
- one primary CTA
- one primary channel
- one supporting channel
- one weekly scorecard

If you cannot define those clearly, the campaign is too vague.

---

## What Counts As Success

Because BuildOS activation is still being repaired, success is not just raw traffic.

Success should be measured across:

1. visibility and relationship movement
2. qualified visits and replies
3. signups from the right segment
4. early activation behavior when available

If downstream activation data is missing, say so and define the closest measurable proxy.

6. `Offer / Angle`
7. `Hero Asset`
8. `Primary CTA`

11. `Target List Shape`
12. `Scorecard`
13. `7-Day Execution Plan`
14. `Stop / Pivot Rules`
15. `Open Questions`

---

## Strong Defaults

Prefer campaigns that:

- target authors, YouTubers, founder-creators, or adjacent scattered operators
- use warmup + DM/email + one proof asset together
- create learnings even if signups stay low
- can be run by one person in 7-14 days

Avoid campaigns that:

- depend on volume
- depend on polished production before shipping
- depend on the homepage doing all the persuasion
- have no path from attention to conversation
